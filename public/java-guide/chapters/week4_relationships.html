1. Inheritance / Interfaces ("Is-A")

This is the strongest relationship. It represents a parent-child relationship.

  • Inheritance: Car extends Vehicle -> Car IS A Vehicle.
  • Interface: Dog implements Animal -> Dog IS AN Animal.

2. Dependency ("Uses-A")

A temporary relationship where one object uses another. Usually passed as a method parameter.

class Printer { void print(Document doc) { System.out.println(doc.getText()); } }
Dependency: Printer 'uses' Document. It doesn't own it. If Printer dies, Document survives.

3. Association ("Has-A")

A relationship where objects are linked. It comes in two flavors:

a. Aggregation (Weak "Has-A")

The child can exist independently of the parent.

class Department { List<Professor> staff; }
Aggregation: Professors exist even if the Department closes.

b. Composition (Strong "Has-A")

The child CANNOT exist without the parent. They live and die together.

class House { private Room room = new Room(); }
Composition: If you destroy the House, the Room is destroyed too.

Seminar: Identifying OOP Components

Let's analyze a snippet of code and identify all the OOP concepts.

// l. Main Class public class ZooApp { public static void main(String[] args) { // c. Object Animal leo = new Lion(); leo.eat(); // k. Polymorphism } } // h. Abstraction abstract class Animal { // g. Encapsulation (private attribute) private String name; // f. Constructor public Animal(String n) { this.name = n; } // e. Method public abstract void eat(); } // i. Inheritance (j. Inheritance - repeated for emphasis) class Lion extends Animal { public Lion() { super("Leo"); } public void eat() { System.out.println("Meat"); } }
l. Main Class: Entry point.
c. Object: 'leo' is an instance.
k. Polymorphism: Calling eat() on Animal type, but Lion's method runs.
g. Encapsulation: 'name' is private.
i. Inheritance: Lion extends Animal.

Task: Create a `Zoo` class (Aggregation) that has a list of `Animal`s. Create a `Keeper` class (Association) that feeds the animals.

b. Using Java List of Objects

Arrays are fixed size. Lists grow dynamically.

import java.util.ArrayList; import java.util.List; List<Animal> animals = new ArrayList<>(); animals.add(new Lion()); animals.add(new Elephant());
Generics: <Animal> ensures only Animals can be added to this list.
