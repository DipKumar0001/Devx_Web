1. Encapsulation & Access Modifiers

Encapsulation is the mechanism of wrapping data (variables) and code acting on the data (methods) together as a single unit.

Access Modifiers

Modifier Visibility
private Only within the same class.
(default) Within the same package.
protected Same package + subclasses.
public Everywhere.

Getters and Setters

Used to protect your data. You make attributes private and provide public methods to access/update them.

public class Person { private String name; // Hidden // Getter public String getName() { return name; } // Setter public void setName(String newName) { if (newName != null) { this.name = newName; } } }
Private Attribute: Cannot be accessed directly (e.g., person.name = "Bob" will fail).
Setter Logic: We can add validation here (e.g., checking for null) to protect the data integrity.

3. Abstraction - Interface

Abstraction is hiding the implementation details and showing only functionality. An Interface is a pure abstract type.

  • Cannot be instantiated.
  • Methods are abstract by default (no body).
  • A class implements an interface.

Seminar - Case Study

Designing a Remote Control System using Interfaces.

// a. Interface interface Device { // b. Interface Methods (Abstract) void turnOn(); void turnOff(); } class TV implements Device { // c. Attributes / d. Properties private boolean isOn; // e. Methods (Implementation) public void turnOn() { isOn = true; System.out.println("TV is On"); } public void turnOff() { isOn = false; System.out.println("TV is Off"); } }
Interface: Defines the contract. Any 'Device' must be able to turn on and off.
Implementation: The TV class provides the actual logic for these methods.

Lab 2: Packages & Encapsulation

i. Understanding Packages and Sub-Packages

Packages are used to group related classes. They help avoid name conflicts and control access.

package com.zoo.animals; public class Lion { ... }
Package Declaration: Must be the first line of the file.

ii. Encapsulating Lion and Elephant

Task: Create classes `Lion` and `Elephant` in a package. Make their attributes private (weight, age) and provide public getters/setters.

iii. Using Interface to contain common methods

Task: Create an interface `AnimalBehavior` with methods `eat()` and `sleep()`. Make `Lion` and `Elephant` implement it.

public interface AnimalBehavior { void eat(); void sleep(); } public class Lion implements AnimalBehavior { public void eat() { System.out.println("Lion hunts and eats."); } // ... implement sleep() }
Polymorphism: We can now treat both Lions and Elephants as 'AnimalBehavior' types.

iv. Adding new classes and methods to Packages

Practice organizing your code. Create a `com.zoo.main` package for your `Main` class, and import the animals.

package com.zoo.main; import com.zoo.animals.Lion; public class Main { public static void main(String[] args) { Lion simba = new Lion(); simba.eat(); } }
Import: Required because Lion is in a different package.
