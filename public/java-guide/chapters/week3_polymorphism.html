1. Inheritance

Inheritance is a mechanism where a new class acquires the properties and behaviors of an existing class.

  • Parent Class (Super Class): The class being inherited from.
  • Child Class (Sub Class): The class that inherits from the parent.
// a. Parent Class class Vehicle { protected String brand = "Ford"; public void honk() { System.out.println("Tuut, tuut!"); } } // b. Child Class class Car extends Vehicle { private String modelName = "Mustang"; }
Parent: Defines common attributes like 'brand'.
Child: 'Car' IS-A 'Vehicle'. It gets 'brand' and 'honk' for free.

2. Polymorphism

Polymorphism means "many forms". It allows us to perform a single action in different ways.

a. Method Overloading (Compile-time)

Same method name, different parameters (number or type) within the same class.

class Calculator { int add(int a, int b) { return a + b; } double add(double a, double b) { return a + b; } }
Overloading: The compiler decides which 'add' to call based on the arguments.

b. Method Overriding (Runtime)

Child class provides a specific implementation of a method that is already provided by its parent class.

class Animal { void sound() { System.out.println("..."); } } class Dog extends Animal { @Override void sound() { System.out.println("Bark"); } }
Overriding: The 'Dog' replaces the generic sound with "Bark".

3. Abstraction

Hiding implementation details and showing only functionality.

Abstract Class Interface
Can have abstract and non-abstract methods. All methods are abstract (until Java 8).
Can have instance variables. Only static final variables (constants).
"is-a" relationship. "can-do" relationship.

Seminar - Case Study

Shape System: Demonstrating Abstract Classes and Polymorphism.

// a. Abstract Class / c. Parent Class abstract class Shape { String color; // b. Abstract Method abstract double getArea(); // Concrete method void setColor(String c) { this.color = c; } } // d. Child Class class Circle extends Shape { double radius; // f. Method Overriding @Override double getArea() { return 3.14 * radius * radius; } // e. Method Overloading (Custom resize) void resize(int factor) { radius *= factor; } void resize(double factor) { radius *= factor; } }
Abstract Class: Cannot be instantiated. 'Shape s = new Shape()' is illegal.
Abstract Method: Forces the child class to provide the logic for 'getArea'.
Implementation: Circle MUST implement 'getArea' to be a valid concrete class.

Lab 3: Abstract Classes & Polymorphism

i. Using Abstract Class as Parent

Task: Extend your Zoo project. Create an abstract class `Animal` that implements `AnimalBehavior` (from Lab 2) but leaves `eat()` abstract.

public abstract class Animal implements AnimalBehavior { protected String name; public abstract void eat(); // Still abstract public void sleep() { System.out.println("Zzz..."); } }

ii. Creating Child Classes

Task: Refactor `Lion` and `Elephant` to extend `Animal`. They must implement `eat()`.

iii. Polymorphism in Child Classes

Task: Create a list of Animals and iterate through them.

List<Animal> zoo = new ArrayList<>(); zoo.add(new Lion()); zoo.add(new Elephant()); for (Animal a : zoo) { a.eat(); // Polymorphism in action! }
Polymorphism: We don't know if 'a' is a Lion or Elephant, but we know it can 'eat'. The correct method is called at runtime.
