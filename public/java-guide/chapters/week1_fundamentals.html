1. Procedural vs Object-Oriented Programming

Procedural Programming is about writing procedures or functions that perform operations on the data.

Object-Oriented Programming (OOP) is about creating objects that contain both data and functions.

Procedural Object-Oriented
Program is divided into functions. Program is divided into objects.
Focus on functions/logic. Focus on data/state.
Less secure (data moves freely). More secure (data is hidden/encapsulated).

2. OOP using Java Language

Java is a strictly Object-Oriented language (mostly). Unlike C++, where you can write procedural code, in Java, everything must be inside a class.

  • Platform Independent: "Write Once, Run Anywhere" (WORA).
  • Robust: Strong memory management and exception handling.
  • Secure: No explicit pointers, runs inside a virtual machine (JVM).

3. Classes vs Objects

// The Blueprint (Class) class Car { String color; } // The Real Thing (Object) Car myFerrari = new Car(); Car myToyota = new Car();
Class: Logical template. Does not consume memory for data yet.
Object: Physical instance. Consumes memory. 'myFerrari' and 'myToyota' are separate objects from the same blueprint.

4. Attributes, Functions, and Constructor

  • Attributes (Fields): Variables that hold the state of the object.
  • Functions (Methods): Code that defines the behavior of the object.
  • Constructor: A special method used to initialize the object.

5. User Story

In Agile development, requirements are often written as User Stories. They describe who wants the feature, what they want, and why.

"As a [Role], I want to [Action], so that [Benefit]."

Example:

"As a Library Member, I want to search for books by title, so that I can find the book I need quickly."

From this story, we can identify potential classes (Member, Book) and methods (search()).

6. Seminar - Case Study

Let's design a simple Student Management System component.

public class Student { // d. Attributes (State) String name; int age; // c. Constructor (Initialization) public Student(String n, int a) { this.name = n; this.age = a; } // e. Method (Behavior) public void study() { System.out.println(name + " is studying."); } } class Main { public static void main(String[] args) { // b. Object (Instance) Student s1 = new Student("Alice", 20); s1.study(); } }
a. Class: The 'Student' class defines the structure.
d. Attributes: 'name' and 'age' store data for each student.
c. Constructor: Sets the initial name and age when 'new Student()' is called.
e. Method: 'study()' is an action the student can perform.
b. Object: 's1' is a specific student named Alice.

IntelliJ IDEA is the most popular IDE for Java development.

2. Hello World in Java

Every Java journey starts here.

public class Main { public static void main(String[] args) { System.out.println("Hello, World!"); } }
main method: The entry point of any Java application.
System.out.println: Prints text to the console.

3. Data Types, Functions, Comments, Control Flow

The building blocks of Java syntax.

// This is a single-line comment int count = 5; boolean isActive = true; if (isActive) { for (int i = 0; i < count; i++) { printNumber(i); } } // Function (Method) void printNumber(int n) { System.out.println(n); }
Data Types: 'int' for numbers, 'boolean' for true/false.
Control Flow: 'if' checks a condition, 'for' loops multiple times.
Function: A reusable block of code.
