1. Event-Driven Programming: Integrating Logic

Real-world apps separate the UI (View) from the Logic (Model/Controller). JavaFX allows us to trigger backend logic from frontend events.

// Backend Logic class Calculator { public int add(int a, int b) { return a + b; } } // Frontend (JavaFX) Button btn = new Button("Add"); btn.setOnAction(e -> { int result = new Calculator().add(5, 10); label.setText("Result: " + result); });
Integration: The UI event handler calls the 'Calculator' class. The UI doesn't know HOW to add, it just asks the backend.

2. UML Sequence Diagrams

Sequence diagrams show how objects interact with each other and in what order.

  • Lifeline: Dashed vertical line representing an object's life.
  • Activation Bar: Thin rectangle on the lifeline showing when the object is active.
  • Messages: Horizontal arrows showing method calls.

Seminar: User Stories to Sequence Diagrams

User Story: "As a user, I want to login so I can access my dashboard."

User -> LoginScreen: enterCredentials(id, pass)
LoginScreen -> AuthService: validate(id, pass)
AuthService -> Database: findUser(id)
Database --> AuthService: UserData
AuthService --> LoginScreen: true
LoginScreen -> Dashboard: show()

Explanation: The diagram proves that 'LoginScreen' needs to talk to 'AuthService', which talks to 'Database'. It helps plan your classes and methods.

Lab 10: Integrating Java Code with JavaFX

Task 1: The Backend

Create a `StudentManager` class with methods:

  • addStudent(String name)
  • getStudentCount()

Task 2: The Frontend

Create a JavaFX window with:

  • A TextField for the name.
  • A Button "Add Student".
  • A Label to show the count.

Task 3: Integration

Inside the button's event handler:

  1. Get the name from the TextField.
  2. Call `studentManager.addStudent(name)`.
  3. Update the Label with `studentManager.getStudentCount()`.
