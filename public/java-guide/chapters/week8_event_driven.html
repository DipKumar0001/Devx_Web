1. Event-Driven Programming

In traditional programming, the flow is determined by the programmer (Step 1 -> Step 2). In Event-Driven Programming, the flow is determined by events (User clicks, Key presses, System signals).

  • Event Source: The object that generates the event (e.g., a Button).
  • Event Listener: The object that waits for the event and handles it.
  • Event Object: Contains details about the event (e.g., which key was pressed).

2. Event-Driven Programming using JavaFX

JavaFX is the standard GUI library for Java. It uses a theater metaphor:

  • Stage: The main window.
  • Scene: The content inside the window.
  • Nodes: Elements like Buttons, Labels, TextFields.
import javafx.application.Application; import javafx.scene.Scene; import javafx.scene.control.Button; import javafx.stage.Stage; public class MyApp extends Application { public void start(Stage stage) { Button btn = new Button("Click Me"); // Handling the event btn.setOnAction(e -> { System.out.println("Clicked!"); }); Scene scene = new Scene(btn, 200, 100); stage.setScene(scene); stage.show(); } }
Lambda Expression: 'e -> ...' is a concise way to define the Event Listener.

3. UML Diagrams

a. Use Case Diagram

Shows what the system does from a user's perspective.

  • Actor: Stick figure (User, Admin).
  • Use Case: Oval (Login, Buy Item).
  • System Boundary: Box around the use cases.

b. Activity Diagram

Shows the flow of control (like a flowchart).

  • Start Node: Black circle.
  • Action: Rounded rectangle.
  • Decision: Diamond (Yes/No).
  • End Node: Black circle with a ring.

Seminar: User Stories to Diagrams

User Story: "As a customer, I want to withdraw cash so that I can buy groceries."

a. Use Case Diagram

[Customer] -- (Withdraw Cash)
(Withdraw Cash) ..> (Check Balance) : <>
(Withdraw Cash) <.. (Print Receipt) : <>

b. Activity Diagram

(Start) -> [Insert Card] -> [Enter PIN] ->
If PIN OK -> [Select Amount] -> [Dispense Cash] -> (End)
If PIN Fail -> [Eject Card] -> (End)

Lab 8: JavaFX Implementation

Task 1: Setup

Ensure JavaFX is configured in your IDE (IntelliJ). You may need to add the JavaFX library to your module path.

Task 2: Login Form

Create a simple GUI with:

  • Two Labels ("User", "Pass").
  • Two TextFields (use PasswordField for password).
  • A Button ("Login").

Task 3: Event Handling

Add an event listener to the Login button. When clicked:

  1. Read the text from the fields.
  2. If user="admin" and pass="1234", print "Success" to the console (or show an Alert).
  3. Else, print "Failed".
